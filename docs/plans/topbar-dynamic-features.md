# Идеи по динамическому Topbar

Текущая реализация `Topbar` имеет статичный набор вкладок (`File`, `Edit`, `Special`), которые лишь частично реагируют на состояние (например, блокировка кнопки `Close Window`, если нет активного окна, или появление специального меню при зажатом Shift).

Чтобы сделать `Topbar` более похожим на классические оконные интерфейсы (как Mac OS), можно внедрить следующие улучшения:

## 1. Специфичные меню для приложений (App-Specific Menus)

В зависимости от того, какое окно сейчас в фокусе, в `Topbar` должны появляться уникальные вкладки:

- **Icon Painter:** вкладки `Tools` (карандаш, заливка, пипетка), `Image` (очистить холст, инвертировать цвета).
- **Dither Studio:** вкладки `Adjust` (настройка контраста/яркости), `Palette` (выбор пресета палитры).
- **Model Viewer:** вкладка `View` (переключение wireframe, сброс камеры).

## 2. Контекстное меню "Edit"

Вкладка `Edit` должна менять свое содержимое в зависимости от контекста:

- **Фокус на рабочем столе (Finder):** текущая опция `Clean Up Icons`.
- **Фокус на приложении (например, Icon Painter):** базовые функции `Undo`, `Redo`, `Copy`, `Paste`, `Clear`.

## 3. Контекстное меню "File"

Вкладка `File` также может быть расширена:

- **В приложениях:** `New`, `Save`, `Export...`, `Close`.
- **В Finder:** `New Folder`, `Get Info` (информация о выделенном файле).

## 4. Новая вкладка "Window" (Управление окнами)

Полезно добавить вкладку `Window` для навигации по открытым окнам, особенно если их много и они перекрывают друг друга.

- **Список открытых окон:** Динамический список всех окон (из `useWindowManager`), где клик делает окно активным (выводит на передний план).
- **Действия:** `Minimize` (свернуть текущее), `Minimize All`, `Close All` (перенести сюда из File/Apple меню).

## 5. Индикация активного приложения

Рядом с меню-логотипом "Apple" (символ ¤) можно выводить название текущего активного приложения жирным шрифтом (например, **Icon Painter** или **Finder**). Это дает пользователю четкое понимание того, к чему относятся текущие меню.

## Архитектурные размышления по реализации

Чтобы не перегружать `Topbar.tsx` логикой всех приложений, стоит использовать подход инверсии контроля:

1. **Менеджер меню (Zustand):** Создать стор `useMenuStore` или расширить `useWindowManager`, куда активное приложение может "регистрировать" свои вкладки при получении фокуса (например, через `useEffect`).
2. **Регистрация вкладок:** Приложение передает массив объектов (название вкладки, список элементов меню и callback'и для действий).
3. **Общение (Связь Topbar -> App):** Когда пользователь кликает по кастомному пункту меню в Topbar, вызывается переданный callback. Чтобы избежать проблем с замыканиями (stale closures) в Zustand, приложения могут использовать механизм событий (например, event emitter `mitt`) или хранить мутабельные ref'ы на актуальные функции, которые вызываются из Topbar.
4. **Сброс при расфокусе:** Если фокус переходит на рабочий стол (Finder), кастомные меню очищаются, и Topbar возвращается к стандартному виду.
